# v2sort - высокопроизводительный прокси чекер и сортировщик

> [!CAUTION]
>
> Стабильная работа не гарантируется, возможны сегфолты, аборты, неожиданное поведение и прочее. Пишите об этом в issues или делайте pull запросы с исправлениями

### Возможности:

1. **Поддержка протоколов xray-core**
   - shadowsocks

   - vless

   - socks

   - vmess

   - trojan

   - http
2. **Гибкая конфигурация и настройка вывода**
3. **Автоматическое параллельное тестирование огромных массивов прокси конфигов через xray-core**
4. **Возможность использовать локальную базу mmdb в качестве провайдера геоданных(только на POSIX, сомнительная функция)**
5. **blazing FAST🔥🔥🔥🔥**

#### Требования:

- компьютер на GNU/Linux x86_64
- интернет

*сборки под windows скоро будут*

#### Флаги CLI:

- **-l, --list** 
  - Источники прокси конфигураций. Могут быть как ссылкой на интернет-ресурсы или же локальными файлами. Примеры: `https://example.com/proxies`, `file:///home/user/proxy.txt`, `C:\proxy.txt`
- **-h, --help**
  - Вывести краткую справку о программе
- **-c, --config**
  - Путь к файлу конфигурации. Обязателен
- **-j, --jobs**
  - Количество рабочих потоков. Максимальное число зависит от конфигурации вашей ОС. Устанавливайте разумное значение, иначе программа может упереться в лимит открытых ресурсов(`ulimit -n` на linux), что приведет в ошибочным результатам тестирования
- **-p, --port**
  - Начальный порт для локального SOCKS5 прокси
- **-w, --wait**
  - Время ожидания инициализации xray, указывается в миллисекундах(1000 мс = 1 с)
- **-T, --timeout**
  - Таймаут при обращении к интернет-ресурсам, в секундах
- **-s, --style**
  - Стиль написания отчетов
  - Допустимые значения
    - human
    - raw
    - json

- **-o, --output**
  - Файл для сохранения отчетов. По умолчанию пишется в консоль

- **-b, --bad**
  - Файл сохранения нерабочих/неисправных прокси

- **-R, --regex**
  - Регулярное выражение для извлечения прокси URL

- **-P,  --proxy_per_test**
  - Количество прокси, проверяемых за 1 запуск xray. Рекомендуется выставлять в соответствии с лимитами вашей ОС, например, 150

- **-F, --fragment**
  - Отвечает за форматирование фрагмента(то есть имени, которое будет отображаться в вашем GUI клиенте). Можно включать данные тестирования
    - %url%
    - %country%
    - %city%
    - %region%
    - %http_code%
    - %total%
    - %total_ms%
    - %speed%
    - %speed_kb%
    - %size%
    - %size_kb%

- **-C, --xray_conf**
  - Сохранить конфигурации xray в указанный файл без запуска сетевых тестов

- **-m, --mmdb**
  - Путь к файлу mmdb, который будет использоваться в качестве провайдера геоданных вместо ipinfo.io

- **-v, --verbose**
  - Включить подробные логи

- **-r, --random**
  - Использовать 1 случайный URl из settings.urls. Используется для быстрого тестирования

- **-4, --ipv4_only**
  - Использовать только ipv4

- **-6, --ipv6_only**
  - Использовать только ipv6


### Конфигурация

**xray.path** - путь к исполнимому файлу xray-core. По умолчанию `/usr/local/bin/xray` 

**xray.stdout** - файл для перенаправления стандартного потока вывода xray. По умолчанию `/dev/null`

 **xray.stderr** - файл для перенаправления стандартного потока ошибок xray. По умолчанию `/dev/null` 

**xray.log.access** - путь к файлу логов доступа xray. По умолчанию stdout 

**xray.log.error** - путь к файлу логов ошибок xray. По умолчанию stdout 

**xray.log.log_level** - уровень логирования xray. Допустимые значения: `"debug"` | `"info"` | `"warning"` | `"error"` | `"none"`. По умолчанию `"warning"` **xray.log.dns_log** - включить логирование DNS-запросов через xray. По умолчанию `false` 

**xray.log.mask_address** - маскировка IP-адресов в логах. Допустимые значения: `"quarter"` | `"half"` | `"full"` 

**settings.urls** - список URL-адресов для тестирования производительности прокси 

**settings.protocols.whitelist** - режим работы фильтра протоколов. Если `true` — разрешены только протоколы из списка, если `false` — запрещены только протоколы из списка

**settings.protocols.list** - список протоколов для фильтрации. Регистронезависимо. Допустимые значения: `ss`, `vmess`, `vless`, `trojan`, `socks`, `http` **settings.countries.whitelist** - режим работы фильтра стран. Логика аналогична `settings.protocols.whitelist` 

**settings.countries.list** - список ISO-кодов стран для фильтрации. Регистронезависимо. Пример: `["RU", "CN", "DE"]` 

**settings.ips.whitelist** - режим работы фильтра IP-адресов. Логика аналогична `settings.protocols.whitelist` 

**settings.ips.list** - список IP-адресов для фильтрации. Пример: `["123.456.789", "101.112.131"]`

*Полный пример конфигурации в файле [v2sort_example.toml](v2sort_example.toml)*

### Использование
```bash
v2sort -c .v2sort -l https://example.com/proxies,/home/user/proxy.txt -j 150 -p 10809 -w 1000 -T 5 -s raw -P 150 -o result.txt -R '(?:^|\s)([a-zA-Z][a-zA-Z0-9+.-]*)://([^\r\n]+)'
```

- `-j 150 -P 150` — 150 потоков, 150 прокси за 1 запуск xray (подбирайте под лимиты вашей системы)
- `-w 1000` — ждать 1 секунду после запуска xray перед тестированием
- `-l` принимает несколько источников через запятую — URL и локальные файлы одновременно

### Сборка

Зависимости:
- cmake >= 3.18
- Boost (json, url, filesystem, thread, log)
- libcurl
- ICU (uc, i18n)
- xray-core (runtime, не требуется для сборки)
- libmaxminddb (опционально, только POSIX-совместимые — включает поддержку mmdb)

CLI11 и toml++ подтягиваются как submodules, поэтому клонировать нужно с `--recurse-submodules`:
```bash
git clone --recurse-submodules https://github.com/mamutrahal269/v2sort.git
cd v2sort
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

Бинарник окажется в `build/v2sort`.
